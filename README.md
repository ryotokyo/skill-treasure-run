## ゲーム概要

矢印キーで 🧑‍💻 を操作し、画面に散らばるスキルアイテムを集めながら、敵を避けて 20 秒間でどれだけ稼げるかを競うブラウザゲームです。

アイテムを取るごとにスコアと年収が増え、集めたスキル数に応じてレベルが上がります。
ゲーム終了時の年収でランクが決まります。

## 遊び方

| 操作 | キー |
|------|------|
| 移動 | 矢印キー（← ↑ ↓ →）または W A S D（斜め移動対応） |
| ゲーム開始 | スペースキー |
| リスタート | R キー（ゲームオーバー後） |
| タイトルへ戻る | T キー（ゲームオーバー後） |

---

## 主な機能

- **20 秒タイムアタック** — 残り時間に応じてアイテムの出現ペースが加速
- **年収システム** — 敵に当たると年収が減る（ライフ制なし）
- **アイテム収集** — 通常（82%）とレア（18%）の 2 種類、最大 7 個同時出現
- **敵キャラ 3 種** — 時間経過で段階的に増加するため、後半は難易度が上がる
- **レベルシステム** — 取得アイテム数に応じて肩書きが変わる（全 5 段階）
- **ランクシステム** — ゲーム終了時の年収でランク判定（全 10 段階）
- **ハイスコア保存** — localStorage で最高年収を記録
- **エフェクト** — アイテム取得時のパーティクル・ダメージ時の画面揺れ

## 学習ポイント

### JavaScript
- `requestAnimationFrame` によるゲームループの仕組み
- Canvas API での図形・絵文字の描画
- 円と円の当たり判定（距離計算）
- `filter` で配列の要素を管理（アイテム・敵・パーティクル）
- ベクトル正規化による斜め移動の等速化
- `localStorage` によるスコア永続化

### Git / GitHub
- `git init` → `git add` → `git commit` → `git push` の基本フロー
- SSH キー設定（個人 GitHub アカウントの分離）
- README.md と設計ドキュメントの整備

### 設計
- 状態管理（TITLE / PLAYING / GAMEOVER）の考え方
- データをテーブル（配列）で定義して拡張しやすくする設計
- HUD・フィールド・エフェクトを関数で分割する構造化
