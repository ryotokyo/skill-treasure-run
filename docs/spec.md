## コンセプト

**「エンジニアがスキルアイテムを集めてお金持ちを目指す、60 秒の短編ゲーム」**

- プレイヤーはエンジニア（🧑‍💻）としてスキルを集め、収入を増やす
- 怠惰・浪費・不安という「エンジニアの敵」を避けながら成長する
- 短時間で繰り返し遊べるカジュアルゲーム設計

---

## 画面仕様

| 項目 | 値 |
|------|----|
| キャンバスサイズ | 480 × 640 px |
| HUD 高さ | 72 px（上部固定） |
| プレイフィールド | HUD 下 8px ～ 画面下 8px |
| 背景 | パステルグラデーション（ピンク→ラベンダー→ミント） |

### 画面遷移

```
TITLE → [スペースキー] → PLAYING → [時間切れ / ライフ 0] → GAMEOVER
GAMEOVER → [R キー] → PLAYING（リスタート）
GAMEOVER → [スペースキー] → TITLE
```

## データ構造の考え方

### 状態管理

```
state = 'TITLE' | 'PLAYING' | 'GAMEOVER'
```

ゲームの現在地を 1 つの変数で管理し、描画・更新処理を切り替える。

### アイテム・敵の管理

```javascript
items   = [ { x, y, emoji, pts, money, age, maxAge, ... }, ... ]
enemies = [ { x, y, emoji, effect, value, vx, vy, ... }, ... ]
```

オブジェクトの配列で管理し、`filter()` で「寿命が尽きたもの」「取得済みのもの」を毎フレーム削除する。

### マスターデータとインスタンスを分離

```javascript
// マスターデータ（変わらない定義）
const NORMAL_ITEMS = [ { emoji, name, pts, money }, ... ]

// スポーン時にマスターデータをコピーして位置などを付与
items.push({ ...def, x, y, age: 0, ... })
```

定義と状態を分けることで、アイテムの種類変更が簡単になる。
